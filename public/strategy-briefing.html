Core Concept

The Conversearchable Agent

The extension sits as a sidebar on any airline website. It doesn't replace the airline's search — it drives it. The agent manipulates the airline's own search forms, reads their results from the page, and presents them conversationally.

✈

How it works on Delta.com

  1. User activates extension — sidebar slides out on Delta's flight search page
  2. Agent reads the page — understands the user is searching JFK → LAX, March 5th
  3. User talks naturally — "find me something earlier" or "what about nonstop only"
  4. Agent drives Delta's UI — changes the search form, submits, reads new results
  5. Agent presents honestly — ranks Delta's results by user's actual priorities with transparent trade-offs
  6. Cross-airline honesty — "Delta has this nonstop at $380, but Southwest has one for $290 that leaves 6 hours later. No assigned seating though."
  7. Booking stays on-site — user clicks through on Delta.com or follows a link to Southwest

The cross-airline comparison (step 6) is the honesty principle as a feature. External APIs like Duffel provide comparison data, but the airline's own site is the primary search engine. Duffel is a convenience, not a dependency — the architecture doesn't couple to it.

Decisions Needed

Open Questions

These decisions shape the extension's architecture and first release. Each one is a trade-off between speed-to-demo and breadth of coverage.

Decision 01

How site-specific should v1 be?

The extension needs to understand each airline's DOM to drive search and read results. How deep do we go on day one?

  • One airline, polished — pick Delta or United, build a deep adapter that really knows the site. Nail the demo. Recommended
  • Generic-ish — try to work across multiple airline sites from day one, accepting it'll be rougher on each
  • One deep + generic fallback — deep adapter for one flagship site, plus best-effort generic mode for others
Decision 02

Which airline site first?

If we go deep on one, which one makes the best demo and has the most forgiving DOM structure?

  • United (united.com) — large market share, modern-ish site, NDC-forward
  • Delta (delta.com) — premium brand, good UX already (shows the agent improving good UX)
  • American (aa.com) — largest US airline by fleet
  • Southwest (southwest.com) — unique model (no seat assignments), high brand loyalty
  • Other?
Decision 03

Where does the LLM run?

The agent needs Claude for reasoning. Where does inference happen?

  • Your backend server — extension calls your API, which calls Claude. You control the experience, but need hosting. Recommended
  • Client-side with user's API key — extension calls Claude directly. No server needed, but users need an Anthropic key (developer audience only)
  • Hybrid — free tier on your server, power users can BYO key for unlimited use
Decision 04

Does the extension need auth?

Auth enables preferences, history, and multi-user — but adds friction to first use.

  • No auth for v1 — anonymous, zero friction. Preferences stored locally. Auth added later for multi-user. Recommended
  • Optional auth — works anonymously, but sign in to save preferences and enable cross-device sync
  • Required auth — needed from the start to track users for the airline sales pitch
Decision 05

What's the demo scenario?

The first public demo needs to be a single compelling story that someone can experience in 60 seconds.

  • "Make this search better" — user is mid-search, opens sidebar, agent instantly improves results
  • "Find me a better option" — user sees expensive flights, agent finds cheaper alternatives across airlines
  • "Plan my whole trip" — user describes the trip in plain English, agent handles everything
  • Something else?
Architecture Note

Duffel Is Not the Foundation

Duffel ($3 + 1% per booking, 300+ airlines, clean TypeScript SDK) is useful for cross-airline comparison data. But the architecture treats it as one adapter behind a FlightSearchProvider interface. The core search capability is the airline's own website, driven by the extension.

If Duffel disappears tomorrow, the core product is unaffected. Cross-airline comparison degrades to "check Southwest yourself" — which is still honest.
